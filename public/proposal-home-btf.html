A. 수익성 증대 · 홈화면 리디자인

홈 BTF를 바꾼다
컨텍스트 재주문 · 큐레이션 · 마감딜 스윔레인 설계

"자주 주문한 이 메뉴 어때요?"를 버린다. BTF 전 구간을 컨텍스트 기반 재주문·마감딜 + 헤비유저 큐레이션 + 블로거 검증 3개 스윔레인으로 교체해 홈 진입이 곧 주문으로 이어지게 한다.

Why — 지금 홈 BTF의 문제
문제 01
맥락 없는
재주문 UI
카테고리 혼합 카드 나열. 저녁에 아침 메뉴, 월요일에 주말 메뉴 노출. 습관 형성이 안 된다.
문제 02
별점 4.8이
너무 많다
상위 가게 대부분이 유사한 별점대. 배민·쿠팡과 동일한 큐레이션 로직. 선택 기준 없음.
문제 03
"뭘 시킬지
모르겠다"
새로운 가게를 발견할 동선이 없다. 홈이 정적이라 재방문 이유가 약하다.

솔루션 — BTF 3개 스윔레인 설계
1

지금 이 순간 재주문 — 컨텍스트 × 마감딜

시간·위치·주기 신호로 "지금 이 사람에게 맞는 가게 하나"를 노출. 마감 임박 재고 할인도 같은 레인에서 시간 컨텍스트로 서핑.
시간 컨텍스트 위치 기반 주기 학습 마감 임박 재고 할인 사장님 자발적 할인

노출 카드 3유형 — 컨텍스트에 따라 자동 선택
유형 A — 습관 재주문
과거 주문 패턴이 반복될 시점에 해당 가게를 단독 노출. 클릭 1번으로 바로 장바구니.
"매주 월 오전, 강남구 오피스 → ☕ 오늘도 아메리카노 어때요?"
유형 B — 버티컬 숏컷
지금 시간대에 가장 많이 주문된 카테고리 하나만 노출. 가게 여러 개가 아닌 버티컬 단위 집중.
"금요일 밤 11시 → 🍗 치킨 지금 배달되는 곳 3곳"
유형 C — 마감딜 카드
마감 2시간 전 재고 처리 할인 중인 가게를 시간 컨텍스트 레인에 편입. 사장님은 광고비 없이 재고 소진, 유저는 할인 혜택.
"오후 9시 → 🥗 마감 1시간 전 샐러드 30% 할인 · 한정 수량"
마감딜이 이 레인에 들어오는 이유: 마감딜은 "지금 이 시간에만 유효한 정보"다. 별도 탭이 아니라 컨텍스트 재주문 레인에 통합하면 사용자가 새로운 탭을 학습할 필요 없이 자연스럽게 소비된다. 사장님 입장에서도 광고비 경쟁 없이 재고 소진 목적으로 자발적 할인을 설정하는 구조 — 할인랭킹과 달리 카니발이 없다.
2

매니아 픽

특정 카테고리 헤비유저의 집단 큐레이션
"치킨왕이 인정한 집"
헤비유저 집단지성 카테고리별 독립 리그
월 N회 이상 주문한 카테고리 헤비유저의 별점 5점 가게만 선별 노출. 단순 평균 별점이 아닌 "이 카테고리를 진짜 많이 먹어본 사람이 인정한 가게"라는 신뢰 레이어 추가. 헤비유저는 큐레이터 정체성을 얻어 리뷰 작성 동기 강화.
3

검증된 맛집

파워블로거 실방문 픽 — 네이버 플레이스
상위 블로거가 직접 가서 확인한 가게
외부 신뢰 레이어 실방문 검증
네이버 플레이스 상위 블로거가 실방문 후 작성한 리뷰를 탐색 영역 특별 콘텐츠로 연동. 요기요 입점 가게는 바로 주문 연결. 배민·쿠팡에 없는 외부 검증 신뢰 레이어 — "블로거가 가봤다"는 한 문장이 별점보다 강하다.

핵심 논리: ATF(검색·배너)는 바꾸기 어렵다. BTF는 지금 당장 바꿀 수 있다. 재주문 UI를 컨텍스트 인식으로 바꾸면 습관이 생기고, 마감딜을 같은 레인에 얹으면 사장님이 자발적으로 할인을 건다. 큐레이션 스윔레인은 별점 없이도 신뢰를 만든다. BTF 3개 스윔레인은 배민·쿠팡에 없는 홈화면 차별화 포인트다.
