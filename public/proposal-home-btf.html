홈 BTF를 바꾼다
컨텍스트 재주문 · 매니아 큐레이션 · 찜 재방문 · 메뉴 발견

📄 제안 제출용 PDF 미리보기

"자주 주문한 이 메뉴 어때요?"를 버린다. BTF 전 구간을 컨텍스트 기반 재주문 + 헤비유저 큐레이션 + 찜 재활성화 + 메뉴 발견 동적 슬롯, 최대 4개 스윔레인으로 교체해 홈 진입이 곧 주문으로 이어지게 한다.

Why — 지금 홈 BTF의 문제
문제 01
"뭘 시킬지
모르겠다"
새로운 가게를 발견할 동선이 없다. 홈이 정적이라 재방문 이유가 약하다.
문제 02
별점 4.8이
너무 많다
상위 가게 대부분이 유사한 별점대. 배민·쿠팡과 동일한 큐레이션 로직. 선택 기준 없음.
문제 03
맥락 없는
재주문 UI
카테고리 혼합 카드 나열. 저녁에 아침 메뉴, 월요일에 주말 메뉴 노출. 습관 형성이 안 된다.
개입 구간 — ATF는 건드리지 않는다, BTF를 바꾼다

1

지금 이 순간 재주문

시간·위치·요일·주기 신호를 조합해 "지금 이 사람에게 맞는 버티컬 하나"만 노출. 카테고리 혼합 카드 나열을 대체한다.

노출 카드 2유형 — 컨텍스트에 따라 자동 선택
유형 A — 습관 재주문
과거 주문 패턴이 반복될 시점에 해당 가게를 단독 노출. 클릭 1번으로 바로 장바구니.
"매주 월 오전, 강남구 오피스 → ☕ 오늘도 아메리카노 어때요?"
유형 B — 버티컬 숏컷
지금 시간대·요일에 맞는 카테고리 하나만 노출. 가게 여러 개가 아닌 버티컬 단위로 집중.
"금요일 밤 11시 → 🍗 치킨 지금 배달되는 곳 3곳"
📌 심층 케이스 — 모닝커피 픽업 패턴

배민 현재
홈에 "최근 주문 가게" 리스트 나열. 시간·위치 맥락 없음. 아침에도 야식 가게가 뜬다.
요기요 모닝커피 픽업
지금 이 순간 맞는 가게 하나만 노출. 픽업 거리·대기시간까지 인라인 표시. 클릭 1번 주문.
핵심: 바니프레소가 커피 단일 브랜드 앱으로 구현한 "오늘도 아메리카노?" 경험을 요기요는 모든 카테고리·전 시간대에서 구현한다. 맥락이 맞을수록 클릭 없이 주문으로 간다.
2

매니아 픽

특정 카테고리 헤비유저의 집단 큐레이션
"치킨왕이 인정한 집"
헤비유저 집단지성 카테고리별 독립 리그
월 N회 이상 주문한 카테고리 헤비유저의 별점 5점 가게만 선별 노출. 단순 평균 별점이 아닌 "이 카테고리를 진짜 많이 먹어본 사람이 인정한 가게"라는 신뢰 레이어 추가. 헤비유저는 큐레이터 정체성을 얻어 리뷰 작성 동기 강화.
3

찜해두고 잊은 가게

찜만 하고 주문 안 한 가게에 변화가 생기면
홈에 다시 등장 — 재방문 이유를 만든다
찜 재활성화 변화 트리거 재방문 루프
찜했지만 주문하지 않은 가게에 할인·신메뉴·재오픈 등 변화가 생기면 BTF에 자동 노출. "네가 찜한 집에 오늘 신메뉴가 생겼어요" — 외부 의존 없이 플랫폼 데이터만으로 작동하는 재방문 트리거. Steam 위시리스트 할인 알림과 동일한 구조.
트리거 조건 — 할인 쿠폰 발행 · 신메뉴 등록 · 휴무 후 재오픈 · 리뷰 급증
찜의 재정의 — 지금까지 찜은 저장만 되고 끝이었다. 트리거가 붙으면 찜이 재방문 채널이 된다.
04
메뉴 추천 동적 슬롯
상황에 따라 다른 메뉴 발견 방식 — 데이터가 없으면 레인 자체 미노출
CASE A
트렌드 메뉴
이번 주 주문 급상승 메뉴를 카드로 노출. 가게가 아닌 메뉴명이 먼저 보인다.
발동 조건
최근 7일 주문 급증 메뉴 ≥ 3개
CASE B
취향 크로스
내가 주문한 적 없는 카테고리인데 비슷한 취향의 사람들이 많이 시킨 메뉴를 추천.
발동 조건
주문이력 ≥ 5건 + 협업 필터링 매칭
CASE C
메뉴 비교
같은 메뉴(예: 김치찌개)를 파는 근처 가게 3곳을 한 화면에서 메뉴 카드로 비교.
발동 조건
동일 메뉴 취급 가게 ≥ 3개 (반경 3km)
동적 슬롯 원칙 — 3개 케이스 중 조건을 충족하는 케이스만 노출. 아무것도 없으면 레인 자체를 숨긴다. 억지로 채우지 않는다.
배민과의 차별점 — 배민은 긍정 리뷰 메뉴를 가게 상세에서 보여준다. 요기요는 홈에서 메뉴가 먼저 등장한다. 가게를 고른 뒤 메뉴를 보는 게 아니라, 메뉴를 보고 가게로 진입하는 역순 탐색.
