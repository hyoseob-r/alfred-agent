CORE CONCEPT

"치킨왕이 인정한 집" —
카테고리 헤비유저의 집단 큐레이션

특정 카테고리를 월 N회 이상 주문한 헤비유저의 별점 5점 가게를 별도 스윔레인으로 노출. 단순 평균 별점이 아닌 "이 카테고리를 진짜 많이 먹어본 사람이 골라준 가게"라는 신뢰 레이어를 추가. 헤비유저는 큐레이터로서 정체성을 얻고, 리뷰 작성 동기가 강화됨.

🥗 샐러드 매니아 픽 🐡 홍어 마니아 추천 🍗 치킨왕 인정 맛집 🍜 라멘 오타쿠 선정
전략 설계 — 4개 축
🎯
차별화 포지셔닝

니치 카테고리 먼저

치킨(볼륨) 대신 샐러드·홍어 등 전문성이 부각되는 니치 카테고리로 베타 론칭. "요기요만 있는 전문 큐레이션" 포지셔닝 확보 후 볼륨 카테고리 확장.

🔍
신뢰도 설계

투명한 선정 기준 공개

"월 ○회 이상 주문한 고객의 별점 기준"을 배지에 명시. 광고 오해 차단. 선정 기준 팝업으로 알고리즘 신뢰 형성. Selective 타입 사용자 확보에 핵심.

🏆
리뷰 생태계

헤비유저 = 큐레이터 정체성

리뷰를 쓴 헤비유저에게 "○○왕" 배지 부여. 내 리뷰가 큐레이션에 반영되는 경험 → 리뷰 작성 동기 강화 → 리뷰 데이터 품질 자강 선순환.

🔀
멀티호밍 유인

요기요를 열어야 할 이유

배민·쿠팡에 없는 카테고리 전문가 큐레이션 → 멀티호밍 유저의 앱 전환 동기. "○○ 카테고리 맛집 찾으면 요기요" 포지셔닝.

⚠️

선결 과제 — 데이터 검증 없이 개발 착수 금지

카테고리별 헤비유저 모수 확인 필수. 지역별 월 N회 이상 주문자가 최소 50명 이상이어야 신뢰도 있는 큐레이션 가능. 헤비유저 별점 분포 vs 일반 유저 별점 분포 비교로 "더 짜게 준다" 리스크도 사전 검증 필요. (Dr. Veritas 권고)

실행 로드맵
PHASE 1 · 1주

사전 데이터 분석

  • 카테고리별 헤비유저 모수 집계
  • 헤비유저 별점 분포 분석
  • 베타 론칭 카테고리 선정 (3~5개)
PHASE 2 · 3주

니치 카테고리 베타

  • 스윔레인 1개 + 투명성 배지 구현
  • 헤비유저 분류 배치 파이프라인
  • CTR·CVR A/B 테스트
PHASE 3 · 검증 후

확장 & 생태계

  • 카테고리 확장 (볼륨 포함)
  • 리뷰어 배지·레벨 시스템
  • 마케팅 캠페인 연동
성공 지표 & 가드레일
PRIMARY KPI

스윔레인 CTR · CVR

헤비유저 픽 스윔레인 vs 일반 스윔레인 전환율 비교. 신뢰 기반 큐레이션의 실제 구매 전환력 검증.

SECONDARY KPI

헤비유저 리뷰 작성률

배지 부여 전후 리뷰 작성 빈도 변화. 리뷰 생태계 자강 효과 측정. 재주문율 변화도 추적.

GUARDRAIL

가게 다양성 지수

스윔레인이 특정 소수 가게 집중 노출로 귀결되지 않도록 HHI 모니터링. 지역·연령 편향 점검.
